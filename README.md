# KWS + Fatigue pipeline: обнаружение ключевой фразы и распознавание утомления

## Общее описание

Данный проект реализует двухэтапный пайплайн для анализа речевых сигналов оператора.

1. **Keyword Spotting (KWS)** — бинарная классификация: определение наличия целевой фразы в 1-секундном аудиофрагменте.
2. **Распознавание степени утомления** — трёхклассовая классификация: отсутствие усталости, слабая усталость, сильная усталость.

Пайплайн поддерживает два режима работы:
- **Режим «a»** — анализ длительной аудиозаписи (разбиение на перекрывающиеся окна, последовательная обработка, досрочная остановка при обнаружении сильной усталости).
- **Режим «b»** — анализ одного 1-секундного фрагмента.

Все модули написаны на Python 3.12 с использованием библиотек TensorFlow, Librosa, NumPy, Scikit-learn, Matplotlib, Seaborn и др.

## Структура репозитория

### 1. Подготовка данных для KWS

| Файл | Назначение |
|------|-------------|
| `vkr_kws_feature_extraction.ipynb` | Извлечение MFCC, дельта- и дельта-дельта-коэффициентов из аудиофайлов, формирование датасета `kws_dataset.npz` (38330 примеров, 101 временной фрейм, 39 признаков). |
| `vkr_kws_feature_selection.ipynb` | Анализ информативности признаков (F-отношение), отбор 9 наиболее значимых MFCC-коэффициентов, сохранение сокращённого датасета `kws_dataset_reduced_9.npz`. |

### 2. Обучение модели KWS

| Файл | Назначение |
|------|-------------|
| `vkr_kws_model_training.ipynb` | Создание гибридной модели KWS (свёртки + BiLSTM), обучение, подбор оптимального порога (0.5654), сохранение модели `best_kws_model_v3.keras`. Включает визуализацию истории обучения, матрицу ошибок, анализ влияния весов ошибок на порог. |

### 3. Подготовка данных для fatigue

| Файл | Назначение |
|------|-------------|
| `vkr_fatigue_preprocessing.ipynb` | Предобработка аудио: шумоподавление (spectral gating), мягкая нормализация RMS, приведение к длительности 1 секунда. |
| `vkr_fatigue_feature_extraction.ipynb` | Извлечение полного набора из 43 временных признаков (MFCC + дельты + F0 + RMS + HNR + ZCR) и 2 статистических (jitter, shimmer). Сохранение `fatigue_all_features.npz` (3364 примера). |
| `vkr_fatigue_feature_selection_18.ipynb` | Отбор 18 наиболее информативных временных признаков (ANOVA + Random Forest), сохранение `fatigue_selected_18_features.npz`. |

### 4. Обучение модели fatigue

| Файл | Назначение |
|------|-------------|
| `vkr_fatigue_model_training_evaluate.ipynb` | Архитектура CNN‑BiGRU, обучение, оценка на тестовой выборке (accuracy 88.5%, macro F1 0.87), визуализация. Сохраняет `best_fatigue_model_18_b2.keras`. |

### 5. Полуконтролируемое дообучение fatigue

| Файл | Назначение |
|------|-------------|
| `vkr_fatigue_semi_supervised.ipynb` | Псевдоразметка неразмеченных аудиофрагментов (≈11000 записей), двухэтапное дообучение модели. Сохраняет `best_fatigue_model_18b2_semi.keras`. Качественный анализ предсказаний (уверенность на очевидных и граничных примерах). |

### 6. Объединённый пайплайн

| Файл | Назначение |
|------|-------------|
| `vkr_kws_fatigue_pipeline_1.py` | Основной модуль с функциями `analyze_long_audio_mode_a` и `analyze_one_fragment_mode_b`. Содержит логику загрузки моделей, скейлеров, индексов признаков, предобработки, инференса. |
| `kws_fatigue_pipeline_console.py` | Консольная обёртка для запуска пайплайна из командной строки. |
| `kws_fatigue_pipeline_notebook.ipynb` | Пример использования пайплайна в Jupyter Notebook с настройками путей, порогов и опций. |
| `requirements.txt` | Список всех зависимостей Python для быстрой установки (`pip install -r requirements.txt`). |

## Установка и запуск

1. Клонировать репозиторий.
2. Установить зависимости:
   ```bash
   pip install -r requirements.txt
Запустить обучение моделей, выполняя ноутбуки в указанном порядке.

Для анализа собственных аудиофайлов:

Указать пути к моделям, скейлерам, индексам признаков в kws_fatigue_pipeline_notebook.ipynb.

Вызвать функцию analyze_long_audio_mode_a или analyze_one_fragment_mode_b.

Либо использовать консольный запуск:

bash
python kws_fatigue_pipeline_console.py --mode a --input "path.wav" --kws-model "best_kws_model_v3.keras" --fatigue-model "best_fatigue_model_18b2_semi.keras"
Основные параметры
KWS порог — 0.5654 (оптимизирован по минимизации суммы FP+FN).

Перекрытие окон в режиме «a» — 0.5 секунды.

Предобработка fatigue — шумоподавление включено, мягкая нормализация (RMS = 0.1).

Досрочная остановка — при обнаружении сильной усталости с уверенностью ≥ 0.5.

Результаты
Модель	Метрика	Значение
KWS (binary)	Accuracy	99.8%
KWS (binary)	Precision / Recall / F1	0.9976
Fatigue (3‑class)	Accuracy	88.5%
Fatigue (3‑class)	Macro F1	0.87
Fatigue (3‑class)	Recall (сильная усталость)	0.92
Полуконтролируемое дообучение повысило уверенность модели на очевидных состояниях и дало более взвешенные решения на границе классов без ухудшения количественных метрик.

Примечания
Все .npz-файлы содержат массивы X_train, y_train, X_val, y_val, X_test, y_test.

Скейлеры (scaler_temporal.pkl, scaler_handcrafted.pkl) и индексы признаков (fatigue_selected_18_indices.npy) необходимы для корректного инференса.

Модели ожидают входные данные формы (batch, 101, 9) для KWS и (batch, 101, 18) для fatigue.
